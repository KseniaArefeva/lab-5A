## Обход бинарного дерева и проверка симметричности

### Подробное описание функций

### Класс TreeNode
Узел бинарного дерева. Хранит три поля:
- `value` – значение узла (число, строка и т.п.).
- `left` – ссылка на левого потомка (или `None`).
- `right` – ссылка на правого потомка (или `None`).

### `inorder(root)`
**Симметричный обход** (левое поддерево → корень → правое поддерево).  
Алгоритм: рекурсивно обходим левое поддерево, затем посещаем корень, затем правое поддерево.  
Возвращает список значений в симметричном порядке. Для бинарного дерева поиска такой обход даёт отсортированную последовательность.

### `postorder(root)`
**Обратный обход** (левое поддерево → правое поддерево → корень).  
Алгоритм: рекурсивно обходим левое поддерево, затем правое, затем посещаем корень.  
Возвращает список значений в обратном порядке. Используется, например, для удаления дерева или вычисления характеристик, зависящих от потомков.

### `preorder(root)`
**Прямой обход** (корень → левое поддерево → правое поддерево).  
Алгоритм: рекурсивно посещаем текущий узел, затем левого потомка, затем правого.  
Возвращает список значений в порядке прямого обхода.

### `level_order(root)`
**Обход в ширину (BFS)**, возвращает узлы по уровням.  
Алгоритм: используется очередь (`collections.deque`). Сначала в очереди корень. На каждом шаге фиксируется количество узлов текущего уровня (длина очереди), затем эти узлы извлекаются, их значения добавляются в список уровня, а их левые и правые потомки помещаются в очередь для следующего уровня.  
Возвращает список списков: каждый внутренний список – значения одного уровня (сверху вниз, слева направо).

### `is_symmetric(root)`
**Проверка симметричности дерева относительно корня**.  
Алгоритм: дерево симметрично, если левое поддерево является зеркальным отражением правого. Вспомогательная рекурсивная функция `is_mirror(left, right)` сравнивает два поддерева: проверяет равенство значений узлов и рекурсивно сверяет левого потомка левого поддерева с правым потомком правого поддерева, а также правого потомка левого поддерева с левым потомком правого поддерева.  
Возвращает `True`, если дерево симметрично, иначе `False`. Пустое дерево считается симметричным.

### Вспомогательные функции для тестов
- `build_sample_tree()` – создаёт обычное дерево с 6 узлами.
- `build_symmetric_tree()` – создаёт зеркально-симметричное дерево.
- `build_asymmetric_tree()` – создаёт несимметричное дерево (нарушена зеркальность).

## Тесты

### Тест 1. build_sample_tree

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

**Результаты работы функций:**

- `preorder`   → [1, 2, 4, 5, 3, 6]
- `inorder`    → [4, 2, 5, 1, 3, 6]
- `postorder`  → [4, 5, 2, 6, 3, 1]
- `level_order` → [[1], [2, 3], [4, 5, 6]]
- `is_symmetric` → НЕТ

---

### Тест 2. build_symmetric_tree

```
         1
       /   \
      2     2
     / \   / \
    3   4  4  3
```

**Результаты:**

- `preorder`   → [1, 2, 3, 4, 2, 4, 3]
- `inorder`    → [3, 2, 4, 1, 4, 2, 3]
- `postorder`  → [3, 4, 2, 4, 3, 2, 1]
- `level_order` → [[1], [2, 2], [3, 4, 4, 3]]
- `is_symmetric` → ДА

---

### Тест 3. build_asymmetric_tree

**Структура дерева:**

```
        1
       / \
      2   2
     / \   \
    3   4   3
```


**Результаты:**

- `preorder`   → [1, 2, 3, 4, 2, 3]
- `inorder`    → [3, 2, 4, 1, 2, 3]
- `postorder`  → [3, 4, 2, 3, 2, 1]
- `level_order` → [[1], [2, 2], [3, 4, 3]]
- `is_symmetric` → НЕТ

---

### Тест 4. Пустое дерево (None)

**Результаты:**

- `preorder`   → []
- `inorder`    → []
- `postorder`  → []
- `level_order` → []
- `is_symmetric` → ДА

## Выводы

- Все три рекурсивных обхода (`preorder`, `inorder`, `postorder`) корректно работают для деревьев любой формы, включая пустые.
- Обход в ширину (`level_order`) правильно выделяет уровни даже для неполных и вырожденных деревьев.
- Функция `is_symmetric` корректно определяет симметричность дерева относительно корня, работая по принципу зеркального сравнения поддеревьев.
- Тесты покрывают четыре случая: обычное дерево, симметричное, несимметричное и пустое.
